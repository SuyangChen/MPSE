The columns of the following table represent (1) the input dataset consists of two or three sets of points in 2D. (2) the output of the algorithm is a 3D dataset and this column shows the 2D projections on the corresponding subspaces. (3) the last column contains a link to the interactive tool, where one can interact with the 3D output of the algorithm; clicking on the eye icons rotates the data to the corresponding two or three view points.


The one-two-three dataset consists of 3 images of digits 1, 2 and 3. For each image, we sample a fixed number of points from each digit. Note that the same set of points appears in all images but the pairwise distances between them is different.

The circle-square-triangle dataset consists of the geometric shapes of a circle, square and a triangle. For each image, we sample a fixed number of points from each shape. Note that the same set of points appears in all images but the pairwise distances between them is different.

The clusters dataset consists of 3 images each containing 2 clusters with the same number of data points in each image. Note that the same set of points appears in all images but the pairwise distances between them is different. The first cluster is indicated by (blue, red) points, the second cluster is indicated by (squares, circles), the third cluster is indicated by (filled, empty) points.

The grid-path dataset is used to demonstrate that our proposed algorithm can also be applied for graph visualizations. To demonstrate such an example we create two graphs with 100 vertices: one square grid and one path. Note that the same set of vertices appears in both graphs but the underlying structure is different.
